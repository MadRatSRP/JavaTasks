Сервис аутентификации
http://{host}:{port}/authentication - основная страница приложения
•  поле «логин»;
•  поле «пароль»;
•  кнопка «Войти»;
•  кнопка «Зарегистрироваться».
«Войти» - осуществляется аутентификация пользователя, переадресация на страницу /user, отображающая информацию о пользователе. При ошибочном вводе уведомить о неправильных логине/пароле.
“Зарегистрироваться” – создание нового пользователя, если логин уже существует, выдать предупреждение.
Аккаунты пользователей должны храниться в БД.

Сокращатель ссылок
http://{host}:{port}/shortener  - основная страница приложения
  • поле ввода «длинной» ссылки;
  • кнопка «Создать короткую ссылку», возвращается страница с адресом «короткой» ссылки
При переходе по «короткой» ссылке, происходит переадресация на «длинную» ссылку. Связка ссылок хранится в бд.
Если короткая ссылка неизвестна - редирект на страницу с ошибкой.

+ минимизировать работу с бд


Балансировщик нагрузки
Сервис принимает все входящие запросы и передает их на исполняющие ноды на такой же url. Список нод хранится в бд, должен актуализироваться без перезапуска основного сервиса. Балансировка по round-robin. Исполняющая нода на любой запрос должна возвращать json с информацией о себе (ip, порт и тд и тп), а также логировать поступающие запросы.
